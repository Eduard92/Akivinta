Devapps RealState — API Documentation

Endpoints

Base URL: /api/realstate

1) GET /api/realstate/properties
 - Query params:
   - page (int)
   - per_page (int)
   - categoria_id
   - ciudad_id
   - disponibilidad
   - nombre (search)
 - Response: JSON with `data` array and `meta` pagination.
 - Each property contains `portada` with `path` and `thumbs` (800 and 400 px).

Example:

curl "https://your-site.test/api/realstate/properties?page=1&per_page=12"

2) GET /api/realstate/properties/{slug}
 - Returns detailed property with `portada` and `galeria` (each with `thumbs`).

Example:

curl "https://your-site.test/api/realstate/properties/mi-propiedad-slug"

3) GET /api/realstate/cities
 - Returns active cities [{id, ciudad}]

4) GET /api/realstate/categories
 - Returns active categories [{id, categoria}]

Authentication

- The API supports a simple API key via header `X-API-KEY` or query param `api_key`.
- Configure the key in your environment: set `REALSTATE_API_KEY` in your `.env` file.
  - If `REALSTATE_API_KEY` is not set, the API is public (development mode).

Example with header:

curl -H "X-API-KEY: YOUR_KEY" "https://your-site.test/api/realstate/properties"

Examples and notes

- Responses include thumbnail URLs generated by October's `getThumb()` helper. You can use the `thumbs` URLs directly.
- Caching: list/detail endpoints use simple cache (60s) and lists 300s for cities/categories. Consider invalidating cache on model updates.
- Security: for production, set `REALSTATE_API_KEY` and consider adding rate limiting.

If you want, puedo:
- devolver thumbs adicionales o diferentes tamaños,
- invalidar cache automáticamente con observers,
- añadir autenticación más robusta (JWT, OAuth) o rate-limiting.
